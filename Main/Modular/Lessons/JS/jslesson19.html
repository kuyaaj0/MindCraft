Errors & Debugging

Lesson 19 – Cyberpunk Code Diagnostics Lab

01. Why Debugging Matters

Even the best coders make mistakes! Debugging is the process of finding and fixing errors in your JavaScript code. Understanding how to interpret errors, log data, and handle issues gracefully will make your code reliable and professional.

02. Handling Errors with try/catch

JavaScript stops execution when an error occurs. But you can “catch” issues using a try...catch block to handle them safely without breaking your entire program. 
<script>
try {
  let result = 10 / userInput; // userInput not defined → error
  console.log(result);
} catch (error) {
  console.log("An error occurred:", error.message);
}
</script>
Output → “An error occurred: userInput is not defined”

03. Using Console Tools

The browser’s Console is your #1 debugging tool. Use console.log() to track data, console.error() for issues, and console.warn() for warnings. 
console.log("Data received!");
console.warn("Low memory detected!");
console.error("System crash – restart module!");
Output → Different messages appear in the console with colorful labels.

04. Common Coding Mistakes

Watch out for these frequent errors while coding:
  • Undefined Variables: Using a variable before declaring it.
  • Typos: Misspelling function or variable names (case-sensitive!).
  • Missing Brackets: Forgetting { } or ( ) in conditions or calls.
  • Wrong Data Types: Adding numbers and strings together incorrectly.
  • DOM Timing: Trying to access elements before the HTML is loaded.
Example → “TypeError: Cannot read property ‘value’ of null”

05. Debugging Best Practices

  • Use console.log() liberally to trace variable values.
  • Fix one error at a time and retest after each change.
  • Keep your functions short and focused for easier debugging.
  • Always read the full error message — it shows file, line, and type.
  • Use browser DevTools’ Sources tab to place breakpoints.

06. Debugging Challenge

Test your patience and skill! Find and fix the error in the code below using DevTools or try/catch. 
<script>
try {
  let x = 5;
  let y = 0;
  if (y === 0) throw "Division by zero!";
  let result = x / y;
  console.log("Result:", result);
} catch (e) {
  console.error("⚠ Error detected:", e);
} finally {
  console.log("Program finished.");
}
</script>
Output → Error handled gracefully: “Division by zero!”
