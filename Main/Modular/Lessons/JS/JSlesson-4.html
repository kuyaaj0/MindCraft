01. What Are Variables?

Variables = containers that store data. Like boxes with labels. You can put data inside, change it, use it anywhere. Every program needs variables! 
// Variables = labeled boxes for data
let studentName = "Jhack";     // Box labeled "studentName" contains "Jhack"
let age = 20;                  // Box labeled "age" contains 20
let isActive = true;           // Box labeled "isActive" contains true

console.log(studentName);      // Use the box contents

02. let - Changeable Variables

let = variables you can CHANGE later. Perfect for counters, scores, temporary data. 
let score = 85;           // Start with 85
console.log(score);         // 85

score = 95;                 // CHANGE it!
console.log(score);         // 95 ‚úÖ WORKS!

score = score + 5;          // Update: 100
console.log(score);         // 100 ‚úÖ PERFECT

03. const - CONSTANT Values

const = variables you CANNOT CHANGE. Perfect for fixed values like PI, MAX_SCORE, student ID. 
const MAX_SCORE = 100;    // Fixed maximum
const PI = 3.14159;       // Math constant
const STUDENT_ID = "CMU001"; // Permanent ID

console.log(MAX_SCORE);   // 100 ‚úÖ WORKS
MAX_SCORE = 200;          // ‚ùå ERROR! Cannot change const
  • Use const for data that NEVER changes
  • Use let for data that CHANGES

04. Variable Naming Rules (camelCase)

Names must: start with letter/$/_, use camelCase, no spaces, no keywords. Good names = self-documenting code!
  • ‚úÖ GOOD NAMES:
    let studentName, totalScore, isActive, $cash
  • ‚ùå BAD NAMES:
    let 1stPlace, my-name, class, Student Name

05. Scope Basics - Where Variables Live

Scope = where variable exists. Global = everywhere, Local = only inside blocks. 
// GLOBAL SCOPE (everywhere)
let globalScore = 90;

function calculateGrade() {
    let localBonus = 10;     // LOCAL SCOPE (only inside function)
    let finalScore = globalScore + localBonus;
    console.log(finalScore); // 100
}
// console.log(localBonus); // ‚ùå ERROR! localBonus doesn't exist here
