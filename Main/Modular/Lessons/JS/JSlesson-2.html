01. Statements (;) - Command Terminator

JavaScript statements are like sentences. Each command ends with ; (semicolon). Think of semicolon as a period that tells browser "this command is finished - next one starts now!" 
// ‚úÖ CORRECT - Each statement ends with ;
let name = "CyberNinja";     // Statement 1
let age = 20;                // Statement 2  
console.log(name);           // Statement 3

// ‚ùå WRONG - Missing semicolons = errors!
let name = "CyberNinja"
let age = 20
console.log(name)

Statements execute line by line. Semicolons = "END OF COMMAND"

02. Code Blocks {} - Group Commands

{} creates a block - group of statements that work together. Used in functions, if statements, loops. NO semicolon after closing }. 
function greet() {           // Block starts {
    console.log("Hello");      // Statement 1
    console.log("World!");     // Statement 2
}                          // Block ends } NO semicolon here!

if (age > 18) {            // Condition block
    console.log("Adult");
}
function cyberAlert() {
  alert("CYBERPUNK!");
  console.log("HACKED");
}

// Block = 2 statements working together!

03. Identifiers - Variable/Function Names

Identifiers = names you create (variables, functions). Rules: Start with letter/$_, then letters/numbers/$_, NO spaces, NO keywords. 
// ‚úÖ VALID IDENTIFIERS

// ‚ùå INVALID - Will CRASH!

05. Complete Code Structure

Real programs combine everything: statements + blocks + identifiers. Use consistent naming (camelCase), proper semicolons, organized blocks. 
// CYBERPUNK STUDENT SYSTEM
let studentName = "Jhack";        // Identifier 1
let studentGrade = 95;            // Identifier 2

function calculateGPA() {         // Block starts
    let total = studentGrade + 5; // Local identifier
    console.log(`${studentName} GPA: ${total}`);
}                                 // Block ends

calculateGPA(); // Call function
