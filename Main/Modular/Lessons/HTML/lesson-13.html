Layout & Containers

LESSON 13 - Cyberpunk Layout Matrix

01. What Are Containers?

Containers like div and section organize your page into logical blocks. Think of them as boxes that hold content and control layout. 
<div class="header"> Logo + Navigation </div>
<div class="sidebar"> Menu Items </div>
<div class="main-content"> Articles </div>
<div class="footer"> Copyright </div>

02. div vs section

Use div for generic containers. Use section for content with headings. Both create layout blocks.
  • DIV = Generic box:
    <div class="card">User profile</div>
  • SECTION = Themed content:
    <section> <h2>About</h2> <p>Details...</p> </section>

03. Complete Page Layout

Modern websites use this 4-part structure: Header, Sidebar, Main Content, Footer. 
<header>Navigation bar</header>
<div class="main-container">
    <aside class="sidebar">Menu</aside>
    <main class="content">Articles</main>
</div>
<footer>Copyright</footer>
HEADER
Navigation + Logo
SIDEBAR
Menu Links
MAIN CONTENT
Articles, posts, main info (takes most space)
FOOTER
Copyright + Links

04. Nesting Containers

Containers can contain other containers. Create cards within sections within pages. 
<section class="projects">
    <div class="project-card">
        <h3>Project 1</h3>
        <p>Description</p>
    </div>
    <div class="project-card">
        <h3>Project 2</h3>
    </div>
</section>
