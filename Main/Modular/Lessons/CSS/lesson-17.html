LESSON 17 - Cyberpunk Data Interface

01. Styling Tables <table>

Tables display data in rows and columns. Remove default ugly borders with border-collapse: collapse and create neon grid lines with custom borders and hover effects for perfect readability. 
<table class="cyber-table">
    <thead>
        <tr>
            <th>Student ID</th>
            <th>Name</th>
            <th>Course</th>
            <th>Status</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>CMU001</td>
            <td>Juan Dela Cruz</td>
            <td>BS Information Technology</td>
            <td>Active</td>
        </tr>
    </tbody>
</table>

02. Styling Forms <form>

Forms collect user input. Style inputs with transparent backgrounds, neon borders, and focus states that glow when clicked. Use consistent padding and labels for clean, professional interfaces. 
<form class="cyber-form">
    <div class="form-group">
        <label>Student ID</label>
        <input type="text" placeholder="CMU001234" required>
    </div>
    <div class="form-group">
        <label>Full Name</label>
        <input type="text" placeholder="Enter your full name">
    </div>
    <div class="form-group">
        <label>Program</label>
        <select>
            <option>BS Information Technology</option>
        </select>
    </div>
    <button class="cyber-button">Register Student</button>
</form>

04. Pro Tips & Best Practices

  • Tables: Always use <thead> for headers and <tbody> for data. Hover effects make scanning data easier.
  • Forms: Use label tags with proper association. Focus states are crucial for accessibility.
  • Never: Skip labels or use tiny unreadable fonts in tables. Always make forms keyboard accessible.
