01. Viewport Meta Tag (MUST HAVE)

<meta name="viewport"> makes your site mobile-friendly. Without it, phones show desktop version (tiny text!). ALWAYS include this tag in <head>. 
<head>
<meta name="viewport" content="width=device-width, initial-scale=1.0">
</head>

02. Mobile-First Approach

Mobile-First = Style mobile first, then add media queries for bigger screens. 70% of web traffic is mobile! Start simple, add complexity later. 
/* MOBILE FIRST (default styles) */
.demo-card {
    width: 100%;  /* Full width on mobile */
    margin-bottom: 20px;
}

/* Tablet and up */
@media (min-width: 768px) {
    .demo-layout { flex-direction: row; }
}

/* Desktop */
@media (min-width: 1024px) {
    .container { max-width: 1200px; }
}

03. Common Breakpoints

Breakpoints = screen size triggers. Use these standard sizes: 320px (phones), 768px (tablets), 1024px (laptops), 1440px (desktops) 
/* Phones */
@media (max-width: 480px) { ... }

/* Tablets */
@media (min-width: 768px) { ... }

/* Laptops */
@media (min-width: 1024px) { ... }

/* Desktops */
@media (min-width: 1440px) { ... }

04. Flexible Units

Use %, vw, vh, rem, em instead of fixed px. They scale automatically! Never use fixed widths/heights for responsive design.
  • width: 100%; ‚Üí Full container width
  • font-size: 2rem; ‚Üí Scales with root font
  • padding: 5vw; ‚Üí 5% of viewport width
  • max-width: 1200px; ‚Üí Never too wide

05. Responsive Images

Responsive images scale perfectly. Use max-width: 100%; and height: auto;. 
img {
    max-width: 100%;
    height: auto;
    border-radius: 12px;
}

/* Modern: srcset for different screen sizes */
<img src="small.jpg" 
     srcset="small.jpg 480w, medium.jpg 768w, large.jpg 1200w"
     sizes="(max-width: 480px) 100vw, (max-width: 768px) 50vw, 33vw">
